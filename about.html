Hanh Linux


Pros and cons

Pros

Lightweight
With core tools, non-multilib libraries, OpenRC are installed in the system, we focus more on system size and speed for experienced and advanced users.

Fast
Hanh Linux uses pacman as the main package manager. pacman installs compiled packages instead downloads packages' source code and compiles them on your computer. This will save your time when upgrading and installing new packages.
Also, OpenRC only mounts important filesystems and starts services. systemd controls everything in the system, even your money and time to solve the problem when it is broken.
Customizable
The installer give you chance to select, remove pre-installed packages (including installer) and install Xorg (no DE/WM install support). We also clean everything that the installer provides but you don't need to use.
Also, there some packages will give you source code only if you want to compile yourself like linux-src
Packages
We use Arch, Artix and our mirrors to help you access more packages. Also, AUR is a great place for you if you want to install some unofficial packages.

Cons

Experience
Even that we have the installer, it still requires some manual work like typing, working with fdisk, etc.
If you want to remove pre-installed packages in the system, you must be careful that your system may unbootable because of what you select
Limited packages
Since we uses non-multilib and only some languages for core libraries in the system, some packages may not work properly.
You need to install the Artix gcc-libs and glibc, also gcc for compling.
Rolling releases
pacman installs the newewst packages available in the repositories. That means if you installs some packages are released for a short time, your system may break
